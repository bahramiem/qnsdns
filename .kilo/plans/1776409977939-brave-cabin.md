# Plan: Fix MTU Upload Measurement

## Problem Statement
MTU upload measurement is producing incorrect (underestimated) results. The issue lies in the QNAME overhead calculation when building the upload MTU test query.

## Root Cause Analysis

### Critical Bug in `build_mtu_test_query` (probe.c:358-378)
The upload MTU branch computes:
```c
size_t base_qname_bytes = 1 + domain_len + 2;  // +2 for separator label
```

This is **incorrect** because it omits the length bytes for each domain label. In DNS QNAME encoding:
- Each label has a 1-byte length prefix
- For domain with N labels: total = `domain_len + N + 1` (terminator)
- The separator adds 2 bytes (len=1, char='x')

**Current formula:** `1 + domain_len + 2 = domain_len + 3`  
**Correct formula:** `2 + (domain_len + label_count + 1) = domain_len + label_count + 3`

**Missing:** `label_count` bytes (typically 3-4 for common domains like "example.com")

This causes the actual packet to be larger than `target_mtu`, resulting in false failures and an underestimated optimal MTU.

### Secondary Issue: Complex Binary Search State Machine
The C implementation (`mtu.c`) uses a complex stateful async binary search with `tested_cache` bitmap and intricate termination logic. The Go reference (`mtu.go:796-892`) uses a cleaner synchronous approach:
1. Test high boundary → if success, return high
2. Test low boundary → if fail, return 0
3. Binary search between low+1 and high-1
4. Track best successful value

While the current C code works, it's more fragile and harder to reason about.

## Exact Code Changes

### Change 1: Fix QNAME Overhead in `client/resolver/probe.c`

**Add helper function before `build_mtu_test_query`:**

```c
static size_t count_domain_labels(const char *domain) {
    if (!domain || !*domain) return 0;
    size_t count = 0;
    const char *p = domain;
    while (*p) {
        const char *dot = strchr(p, '.');
        if (!dot) dot = p + strlen(p);
        if (dot > p) count++;
        if (!*dot) break;
        p = dot + 1;
    }
    return count;
}
```

**Replace lines 358-362 in `build_mtu_test_query` upload branch:**
```c
// OLD:
size_t base_qname_bytes = 1 + domain_len + 2;  /* +2 for separator label */
size_t overhead = 12 + 4 + 11 + base_qname_bytes;

// NEW:
size_t domain_qname_size = domain_len + count_domain_labels(domain) + 1; /* +1 for terminator */
size_t base_qname_bytes = 2 + domain_qname_size;  /* 2 for separator (len=1, 'x') */
size_t overhead = 12 + 4 + 11 + base_qname_bytes;
```

This is the **minimum viable fix**. It corrects the packet size calculation without altering any other logic.

### Change 2: Optional Binary Search Refactor (Separate PR)

The binary search in `mtu.c` can be simplified to match the Go reference. This is **not required** to fix the upload MTU measurement bug but improves maintainability.

**New algorithm for `get_next_mtu_to_test` and `mark_mtu_tested`:**
- Remove `tested_cache` (complexity source)
- Use simple interval binary search: `[low, high]` where `low` is last success, `high` is first failure - 1
- Termination: when `low + 1 == high`, optimal = `low`
